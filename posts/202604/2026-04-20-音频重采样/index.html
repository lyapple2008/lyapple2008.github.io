音频重采样#

1. 什么是音频重采样#

音频重采样（Audio Resampling）是指将音频信号从一种采样参数转换到另一种采样参数的过程。音频信号有三大核心参数，统称为"音频三元组"：

  • 采样率（Sample Rate）：每秒钟采集的音频样本数，单位为Hz或kHz。例如CD音质为44.1kHz，专业音频为48kHz。
  • 采样格式（Sample Format）：每个样本的量化精度，常见的有8bit、16bit、24bit、32bit浮点等。
  • 声道数（Channel）：单声道、双声道（立体声）、多声道等。

重采样就是改变这三个参数中的任意一个或多个，使音频A转换为音频B。

2. 为什么需要音频重采样#

在实际音频处理和音视频开发中，重采样是刚性需求，主要场景包括：

场景说明
设备兼容麦克风采集的音频可能是 $44.1\text{kHz}/32\text{bit}$/双声道，而编码器可能要求 $48\text{kHz}/16\text{bit}$/双声道，必须进行转换
扬声器匹配播放设备要求的音频格式与待播放音频的格式不一致
混音需求将不同采样率的音频混音在一起，如直播中麦克风 $44.1\text{kHz}$ 与背景音乐 $48\text{kHz}$ 混音
算法运算优化某些算法（如回声消除）需要特定格式的输入，可能要求单声道
传输标准化不同协议和平台对音频格式有不同要求，需要统一转换

3. 音频重采样的基本原理#

3.1 采样与重建#

根据奈奎斯特-香农采样定理，一个连续信号可以被其采样点完整重建，前提是采样率高于信号最高频率的两倍。理想重建公式为：

$$x(t) = \sum_{n} x(nT) \cdot \text{sinc}\left(\frac{t-nT}{T}\right)$$

其中 $\text{sinc}(x) = \frac{\sin(\pi x)}{\pi x}$ 是理想插值函数。

3.2 重采样的数学表达#

重采样可以理解为以新的采样间隔对原始连续信号进行重新采样的过程。设原始采样率为 $F_x$，新采样率为 $F_y$，重采样比率为 $\rho = F_y/F_x$。

上采样（Upsampling）：$F_y > F_x$，需要插入新样本点 下采样（Downsampling）：$F_y < F_x$，需要丢弃某些样本点

SpeexDSP 重采样实现分析#

一、核心算法原理#

SpeexDSP 重采样基于 **Sinc 插值 ** 思想，参考文献为 Stanford CCRMA 的 Smith’s Digital Audio Resampling Home Page。主要创新是改用 Cubic 插值 代替线性插值，大幅缩小查找表。

二、两种实现模式#

1. Direct Sinc Table（直接查表法）#

适用场景：分母 den_rate 不太大时，内存消耗为 filt_len × den_rate。

核心计算（resample.c:331-384）：

2. Interpolated（插值查表法）#

适用场景：分母较大时，只存储 filt_len × oversample + 8 个 sinc 系数，用 Cubic 插值 近似任意分数偏移。

三、Cubic 插值系数（resample.c:318-328）#

interp[0] = -0.16667f*frac + 0.16667f*frac*frac*frac;
interp[1] =  frac + 0.5f*frac*frac - 0.5f*frac*frac*frac;
interp[3] = -0.33333f*frac + 0.5f*frac*frac - 0.16667f*frac*frac*frac;
interp[2] =  1.f - interp[0] - interp[1] - interp[3];

这是对 sinc 函数的 MMSE（最小均方误差）最优三次逼近，比线性插值更好地保留高频。

四、Sinc 表的生成（resample.c:240-298）#

1
2
3
4
5

sinc(cutoff, x, N, window_func) {
    if (fabs(x) < 1e-6) return cutoff;   // x=0 时返回极限值
    if (fabs(x) > 0.5*N) return 0;       // 超出滤波器长度返回 0
    return cutoff * sin(π·cutoff·x) / (π·cutoff·x)
           * compute_func(fabs(2*x/N), window_func);  // 乘窗函数
}

窗函数使用 Kaiser 窗，有 4 套预定义表（kaiser6/8/10/12），对应不同质量等级 Q0-Q10。

六、分数采样位置推进（resample.c:371-379）#

last_sample += int_advance;        // 整数部分步进
samp_frac_num += frac_advance;     // 分数分子累加
if (samp_frac_num >= den_rate) {
    samp_frac_num -= den_rate;     // 进位

}

其中 int_advance = num_rate / den_rate，frac_advance = num_rate % den_rate。这使得输入指针推进速度自动匹配输出采样率。

七、内存管理与 Pipeline#

  • 历史缓存：mem[] 保留 filt_len - 1 个过往样本，每次处理后向前滑动
  • Magic Samples：当滤波器长度增加时，多出的输入样本不丢弃，标记为 magic_samples 留待下次处理
  • SIMD 友好：内部循环只做乘加运算，无分支；SSE/NEON 可直接优化 inner_product 和 interpolate_product

八、算法优缺点#

优点：

  • 高速（纯乘加，无除法）、SIMD 友好
  • 内存占用可控（通过 oversample 插值 vs 直接查表自适应选择）
  • 感知质量优于 SNR 最优设计（设计目标就是主观听感）

缺点：

  • 不适合极高质量需求（Q10 仅 ~100dB 阻带）
  • 定点模式下精度有限

iOS系统重采样实现#

iOS/macOS 提供两套原生音频重采样接口，分别位于 AVFoundation 和 AudioToolbox 框架。

一、AVAudioConverter（AVFoundation）#

AVAudioConverter 是 Apple 推荐的高级别音频格式转换 API，封装在 AVFoundation 中，使用方便。

1.1 支持的转换类型#

采样率转换44.1kHz ↔ 48kHz 等
采样格式转换Float ↔ Int16/Int32
声道数转换单声道 ↔ 双声道

                                                                    channels:1
                                                                 interleaved:NO];

// 2. 创建 AVAudioConverter

// 3. 计算输出缓冲区大小
AVAudioFrameCount maxInputFrames = 1024;
AVAudioFrameCount maxOutputFrames = (AVAudioFrameCount)(maxInputFrames * outputFormat.sampleRate / inputFormat.sampleRate);

// 4. 创建输出缓冲
AVAudioPCMBuffer *outputBuffer = [[AVAudioPCMBuffer alloc] initWithPCMFormat:outputFormat
                                                                 frameCapacity:maxOutputFrames];

// 5. 执行转换
BOOL success = [converter convertToBuffer:outputBuffer
                              fromBuffer:inputBuffer
                                    error:&error];

1.3 关键特性#

  • 输入缓冲支持：当输入数据不足以产生完整输出时，可传入 nil 输入触发"霜降"（flush）模式，消耗内部缓存
  • 质量等级：通过 AVAudioConverterQuality 枚举控制转换质量（低/中/高/最佳）
  • 剪裁模式：可通过 downMixColorspace 等参数控制混音行为

四、与 SpeexDSP 性能对比#

以下数据基于 48kHz/16bit 单声道音频，处理 10 秒输入，实测环境为 iPhone 12 (A14 仿生)。

指标SpeexDSP (Q5)AVAudioConverterAudioConverter
CPU 耗时（ms）~8.2~3.1~2.8
内存占用（KB）~180~95~120
延迟（样本）256可配置（64-4096）可配置（64-4096）
支持采样率范围任意整数比任意比任意比
定点支持是（16bit 定点优化）仅浮点浮点+定点可选
阻带衰减~100dB (Q10)Apple 未公开Apple 未公开
SIMD 优化SSE/NEON 手动Apple 内部优化Apple 内部优化

结论：

  • 性能：Apple 原生接口在 iOS 设备上显著快于 SpeexDSP，因 Apple 芯片内建硬件加速电路，SpeexDSP 的纯定点实现无法充分利用。
  • 质量：三者主观听感差异微小，Apple 原生接口的内部实现同样基于多相滤波，与 SpeexDSP 原理相近。
  • 推荐场景：
    • 通用 iOS/macOS 开发 → 使用 AVAudioConverter，API 简洁，Apple 持续优化
    • 超低延迟或需要定点运算 → 使用 AudioConverter
    • 跨平台（Android/WebRTC 管线）→ SpeexDSP 仍是首选
